# Contributing to Snekbooru

First off, thank you for considering contributing to Snekbooru! It's people like you that make Snekbooru a great tool for the community.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and inclusive environment. 

## How Can I Contribute?

### Reporting Bugs
Before creating bug reports, please check our [Discord server](https://discord.gg/BqNxn7ftqn) to see if the issue has already been reported. When reporting a bug, please include:
*   Your operating system and version.
*   Steps to reproduce the bug.
*   Expected behavior vs. actual behavior.
*   Screenshots or logs if possible.

### Suggesting Enhancements
Feature requests are welcome! Please share your ideas in the `#suggestions` forume on our Discord or open a GitHub Issue with the correct label.

### Your First Code Contribution
1.  Fork the repository.
2.  Create a new branch for your feature or bug fix.
3.  Implement your changes.
4.  Ensure your code follows our standards (see below).
5.  Submit a Pull Request.

## Style Guidelines

### Python Coding Standards
*   **Minimalism**: We prefer clean, self-documenting code. Avoid excessive comments and verbose docstrings unless the logic is exceptionally complex.
*   **Consistency**: Follow the existing code structure.
*   **Type Hinting**: Use type hints where possible to improve code clarity and IDE support.
*   **Dependencies**: If you add new dependencies, ensure they are added to `requirements.txt`, although we are less likely to approve heavy dependencies, or ones that can be easily replaced with native Python code.

### Pull Request Process
1.  Update the README.md with details of changes if applicable.
2.  The PR will be reviewed by the maintainers.
3.  Once approved, it will be merged into the main branch.
